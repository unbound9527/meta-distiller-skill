# 维度：系统设计直觉与架构思维

## 提取提示词
从技术决策和架构讨论中提炼该目标的系统思维。分析：
1. 他评估一个系统方案的核心标准是什么？
2. 他如何处理复杂度（分而治之/抽象/延迟决定）？
3. 他在架构决策中的典型权衡取舍是什么（性能vs可维护性）？
4. 他如何预见未来的扩展需求？

- 核心评估标准：{描述}
- 复杂度处理策略：{描述}
- 典型权衡取舍：{描述}
- 扩展性考量：{描述}
