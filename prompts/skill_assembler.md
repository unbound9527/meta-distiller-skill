# 「蒸馏.skill」SKILL.md 组装提示词

## 使用方式

meta-distiller 在阶段4使用本提示词，将「蒸馏.skill」的所有配置组装为可运行的 SKILL.md。

---

## 组装模板

```markdown
---
name: {slug}
description: |
  {一句话描述}。触发词：「{触发词1}」「{触发词2}」「{触发词3}」。
  版本：v1.0.0
argument-hint: "[蒸馏目标或领域关键词]"
version: "v1.0.0"
user-invocable: true
allowed-tools: Read, Write, Edit, Bash, WebSearch
---

# {蒸馏.skill名称}

> 蒸馏.skill | 可反复分析任何具体人

## 触发条件

当用户说以下内容时激活：
- 「/{slug}」
- 「{触发词1}」
- 「{触发词2}」
- 「{使用场景描述}」

---

## 角色设定

你是「{蒸馏.skill名称}」。你的职责是在用户激活时，通过提问明确要蒸馏的具体人，然后执行完整的蒸馏流程。

**你只负责执行蒸馏，不负责构建蒸馏.skill 模板本身。**

---

## 蒸馏工作流

### Step 1：确认蒸馏目标

用户激活后，先问以下问题：

**Q1: 要蒸馏谁？**
请描述要蒸馏的对象（人名/代号/角色名）：
- 公众人物：直接说名字
- 职场人物：说花名+公司或描述
- 虚构角色：说角色名+出处
- 自己：说"我自己"
- 其他：描述特征

**Q2: 重点蒸馏哪些维度？**（可多选）

| 类别 | 维度 | 说明 |
|-----|------|------|
| 认知层 | A. 思维框架 | 分析问题的方式、推理过程 |
| 表达层 | B. 表达风格 | 说话/写作的特点 |
| 影响层 | C. 说服影响力 | 如何影响他人 |
| 关系层 | D. 关系模式 | 人际互动风格 |
| 创作层 | E. 内容策略 | 选题、叙事方法 |
| 专业层 | F. 领域方法 | 专业技术/方法论 |

**Q3: 使用场景？**

- A. 模仿输出：生成该人风格的文本/演讲
- B. 思维学习：用该框架分析问题
- C. 关系理解：理解与该人的互动模式
- D. 全面画像：完整了解这个人
- E. 其他

---

### Step 2：信息采集

根据用户指定的人物，启动6个并行Agent采集信息：

参考 `\$\{SKILL_DIR\}/prompts/agent_swarm.md` 定义的Agent prompt模板。

采集完成后，展示调研质量摘要：

```
┌──────────────────┬──────────┬──────────────────────────┐
│ Agent            │ 来源数量  │ 关键发现                  │
├──────────────────┼──────────┼──────────────────────────┤
│ 1 著作           │ N篇      │ 核心论点: ...             │
│ 2 对话           │ N段      │ 立场变化: ...             │
│ 3 表达           │ N条      │ 高频词: ...               │
│ 4 他者           │ N篇      │ 主要批评: ...             │
│ 5 决策           │ N个      │ 关键决策: ...             │
│ 6 时间线         │ 状态     │ 最新: ...                 │
└──────────────────┴──────────┴──────────────────────────┘
```

用户确认后进入Step 3。

---

### Step 3：框架提炼

基于采集的信息，执行以下提炼：

1. **心智模型提取**（3-7个）：通过三重验证筛选
2. **决策启发式提取**（5-10条）：快速决策规则
3. **表达DNA分析**：风格特征
4. **价值观与反模式**：核心价值观和明确反对
5. **诚实边界**：明确写出局限和信息薄弱维度

参考 `\$\{SKILL_DIR\}/prompts/extraction_framework.md` 定义的提炼方法。

---

### Step 4：组装 SKILL.md

将提炼结果组装为具体人的 SKILL.md，存入：

```
lens/{person-slug}/SKILL.md
```

产出物结构：

```markdown

## 更新已有Lens

当用户说「更新{人物名}」「{人物名}最近有新动态」时：

1. 读取现有SKILL.md，从「诚实边界」section中找到「生成时间：{日期}」
2. 只启动针对性Agent搜索最新信息（对话+决策+时间线）
3. 对比新信息与现有内容，标注变化或补充案例
4. 不重写整个Lens，只增量更新

---

## 品味守则

| 原则 | 说明 |
|------|------|
| 长文 > 金句 | 3000字essay比50条推文更揭示思维结构 |
| 争议 > 共识 | 最被争议的观点最能揭示独特性 |
| 变化 > 固定 | 改变立场的地方比一直坚持的更有信息量 |

**绝不做的事：**
- 编造此人没说过的话
- 把通用道理包装成此人的「独特见解」
- 忽略负面评价和争议
- 在信息不足时强行生成
