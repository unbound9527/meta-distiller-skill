# celebrity-distiller

名人口袋分身 | 学习名人的思维框架和表达习惯

---

## 执行流程

蒸馏过程分为三个阶段：

### 阶段 1：语料采集
1. 根据用户指定的名人，搜索并采集公开语料
2. 按优先级排序：深度访谈 > 亲笔文章 > 社交媒体 > 第三方报道
3. 语料预处理：去重、分块、标注来源
4. 输出：corpus/{person-slug}/raw/

### 阶段 2：维度分析
1. **D1 思维框架（Logic CoT）**：如何思考问题、推理过程、决策框架
2. **D2 表达风格（Persona）**：语言特征、叙事策略、人设标签
3. **D3 说服影响力（Influence Pattern）**：受众定位、影响策略、传播设计

### 阶段 3：Lens 组装
1. 提炼核心发现
2. 生成可激活的 SKILL.md
3. 输出到 lens/{person-slug}/

## 语料采集策略

### 采集优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| P0 | 深度访谈/演讲全文 | YouTube视频字幕、播客全文、会议演讲 |
| P1 | 亲笔文章/书籍 | 官方博客、专栏、著作、公开信 |
| P2 | 社交媒体发帖 | Twitter/X、微博、微信公众号 |
| P3 | 第三方采访 | 新闻专访、人物报道、百科 |

### 最低语料要求

- 有效内容时长 ≥ 5小时（音视频转文字）
- 或文字内容 ≥ 10万字
- 覆盖时间跨度 ≥ 2年

### 采集命令

使用 `tools/corpus_collector.py`：

```bash
python tools/corpus_collector.py --name "埃隆·马斯克" --output corpus/elon-musk/raw/
```

## 维度分析规范

### D1: 思维框架（Logic Chain-of-Thought）

```markdown
## D1: 思维框架

### 问题定义模式
- 如何切入问题？
- 喜欢归因到什么层面？（个人/系统/运气）

### 推理风格
- 演绎 vs 归纳
- 直觉优先 vs 数据优先
- 第一性原理 vs 类比推理

### 决策框架
- 风险偏好
- 确定性要求
- 决策速度

### 思维DNA
- 高频逻辑词：[列表]
- 典型句式：[3-5个代表性句子]
```

### D2: 表达风格（Persona）

```markdown
## D2: 表达风格

### 语言特征
- 词汇偏好：[高频名词、动词、形容词]
- 句式特点：[短句/长句/复合句偏好]
- 节奏感：[快/慢/张弛有度]

### 叙事策略
- 故事类型：[励志/危机/情怀/反叛/...]
- 案例使用：[偏好使用哪类案例]
- 情绪调动：[如何引起共鸣]

### 人设标签
[3-5个关键词：如"激进""乐观""颠覆者"...]

### 表达禁区
[刻意回避的内容或词汇]
```

### D3: 说服影响力（Influence Pattern）

```markdown
## D3: 说服影响力

### 核心受众
[人群画像：年龄/职业/价值观]

### 影响策略
- 理性说服：数据/逻辑/证据
- 情感共鸣：故事/情怀/身份认同
- 权威背书：专家/名人/机构
- 社群营造：粉丝团/会员/社区

### 传播设计
- 如何让信息自传播？
- 制造冲突/反差/惊喜？

### 反说服机制
[面对质疑时的应对方式]
```

## 纠偏机制

当用户说"这个不对"、"维度不对"、"问题应该..."时：

1. 识别纠正是针对哪个维度（D1/D2/D3）
2. 记录纠正内容到 `lens/{person-slug}/corrections.md`
3. 更新 `meta.json` 的 `corrections_count`
4. 重新生成该维度的分析
