# methodology（方法论维度）

## 维度定义

提取专业领域/流派的系统性方法论、操作流程和实践体系。

## 提取要点

### 1. 核心流程
- 完成某类任务的标准步骤
- 方法论的核心框架
- 各阶段的输入输出

### 2. 操作技巧
- 实践中常用的具体技巧
- 关键的判断点/决策点
- 常见的陷阱和如何避免

### 3. 工具和模板
- 方法论配套的工具
- 常用的模板/清单
- 推荐的学习资源

### 4. 适用条件
- 什么情况下适合使用这个方法论？
- 什么情况下需要调整或放弃？
- 与其他方法论的配合

### 5. 效果评估
- 如何判断方法论执行的效果？
- 成功的标志是什么？
- 失败的信号是什么？

## 语料来源

- L1: 方法论文章/教程
- L2: 书籍/课程内容
- L3: 实践案例/文档
- L6: 用户实践经验描述

## 提取提示词

```
请分析以下语料，提取该领域的方法论体系：

1. **核心流程**：完成这类任务的标准步骤是什么？
2. **操作技巧**：实践中有什么关键技巧和常见陷阱？
3. **工具模板**：有什么配套的工具和模板？
4. **适用条件**：什么情况下适合用这个方法论？
5. **效果评估**：如何判断方法论执行的效果？

---
语料内容：
[粘贴方法论相关的教程/实践描述]
```
