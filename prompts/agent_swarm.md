# 「蒸馏.skill」Agent Swarm 定义

> **注意**：meta-distiller 不执行任何实际调研。本文件定义的是「蒸馏.skill」激活时，6个并行 Agent 的 prompt 模板。

## 启动方式

使用 `Agent` 工具启动 6 个并行 subagent，每个 subagent 独立执行调研任务。

```
spawn 6 subagents in parallel
```

## 每个 Agent 的标准 prompt 模板

### Agent 1：著作调研

```
你的任务：调研[目标人物名]的著作和系统性长文。

目标目录：lens/{person-slug}/references/research/01-writings.md

调研方向：
- 此人出版的书籍（书名、核心论点、出版年份）
- 长篇 newsletter/博客/论文
- 反复出现≥3次的核心理念（这些是真信念）
- 自创术语和概念
- 推荐书单（揭示智识谱系）

输出要求：
- 写入目标文件路径
- 每条信息标注来源URL和可信度
- 区分一手（此人写的）vs 二手（别人总结的）
- 发现矛盾直接记录，不要调和

信息源黑名单：知乎、微信公众号、百度百科。
```

### Agent 2：对话调研

```
你的任务：调研[目标人物名]的长对话和即兴思维。

目标目录：lens/{person-slug}/references/research/02-conversations.md

调研方向：
- 播客访谈、YouTube 长视频、AMA
- 被追问时的回答方式（如何展开话题、如何回避敏感问题）
- 即兴类比和解释方式
- 改变立场的瞬间
- 拒绝回答的问题（揭示边界）

输出要求：
- 写入目标文件路径
- 每条信息标注来源
- 区分一手（对话原文）vs 二手（转述）
- 发现矛盾直接记录

信息源黑名单：知乎、微信公众号、百度百科。
```

### Agent 3：表达DNA调研

```
你的任务：调研[目标人物名]的碎片表达和语言风格。

目标目录：lens/{person-slug}/references/research/03-expression-dna.md

调研方向：
- Twitter/X、微博、即刻等短文本平台
- 高频用词和句式（提取 10+ 个标志性表达）
- 争议立场的表达方式
- 幽默和讽刺的使用方式
- 公开辩论或反驳的风格

输出要求：
- 写入目标文件路径
- 列出高频词汇/短语（≥10个）
- 标注表达风格特征
- 区分此人原创 vs 引用

信息源黑名单：知乎、微信公众号、百度百科。
```

### Agent 4：他者视角调研

```
你的任务：调研他人对[目标人物名]的评价和观察。

目标目录：lens/{person-slug}/references/research/04-external-views.md

调研方向：
- 书评、人物评论、第三方分析
- 主要批评和争议点
- 与同行/同领域人物的对比
- 外部观察到的思维模式和行为特征
- 传记或纪录片中的他人叙述

输出要求：
- 写入目标文件路径
- 区分正面评价 vs 批评 vs 争议
- 标注每个观点的来源
- 不要只汇总正面评价

信息源黑名单：知乎、微信公众号、百度百科。
```

### Agent 5：决策调研

```
你的任务：调研[目标人物名]的重大决策和行为模式。

目标目录：lens/{person-slug}/references/research/05-decisions.md

调研方向：
- 人生/职业重大转折点
- 关键商业或创作决策及背景
- 决策时的判断标准（他看重什么）
- 事后反思和复盘（如有）
- 言行一致 vs 言行不一的案例

输出要求：
- 写入目标文件路径
- 每个决策包含：背景、选项、最终选择、结果
- 标注决策背后的逻辑（推断）
- 识别重复出现的决策模式

信息源黑名单：知乎、微信公众号、百度百科。
```

### Agent 6：时间线调研

```
你的任务：构建[目标人物名]的完整时间线。

目标目录：lens/{person-slug}/references/research/06-timeline.md

调研方向：
- 出生/出道到现在的完整时间线
- 每个阶段的关键里程碑
- 思想/风格的转变节点
- 最近 12 个月的最新动态（防止信息过时）
- 重要事件与外部环境的关联

输出要求：
- 写入目标文件路径
- 时间线格式清晰（年份 + 事件 + 意义）
- 标注信息来源
- 最新动态单独标注日期

信息源黑名单：知乎、微信公众号、百度百科。
```

## 执行约束

1. **并行执行**：6 个 Agent 同时启动，互不等待
2. **超时处理**：单个 Agent 5 分钟无有价值结果 → 继续推进，标注「信息不足」
3. **文件写入**：每个 Agent 必须将结果写入指定文件，不存文件的调研等于没做
4. **来源标注**：每条信息必须标注来源和质量等级

## Agent 间协调

- 不需要协调，各自独立调研
- 矛盾点保留，不和稀泥
- 完成后汇总到蒸馏.skill 的调研检查点
